# Адаптация книги Тихонова под URAN

## Что уже готово
- Книга распарсена в структуру `json` и `jsonl` (папка `Uran/тихонов`).
- Ассистент URAN теперь умеет читать `Tikhonov_Aptechnaya_tekhnologia_chunks.jsonl` напрямую (без отдельной переконвертации).

## 1) Адаптация для ассистента (поисковый слой)
- Базовый режим: полнотекстовый retrieval по чанкам книги.
- Формат для ассистента: `title/body/tags/doc_id` уже маппится из текущего `jsonl`.
- Рекомендовано улучшить качество поиска:
  - убрать служебные куски (`[pic]`, большие таблицы без текста);
  - добавить тематические теги: `POWDERS`, `SOLUTIONS`, `OINTMENTS`, `SUPPOSITORIES`, `STERILITY`, `FILTERING`;
  - сделать отдельный `assistant_nlp_assets_pharma.json` с фарм-синонимами (вместо общедоменного шума).

## 2) Адаптация для движка (RxEngine)
Важно: книгу нельзя напрямую превращать в автоправила без валидации. Нужен слой "кандидат -> валидированное правило".

### Предложенная схема
1. Извлечь из книги кандидатные технологические утверждения (предложения с маркерами: "растворить", "нагревать", "фильтровать", "довести до объема", "стерильно").
2. Нормализовать в промежуточный JSON (`engine_rule_candidates.json`).
3. Провести ручную валидацию и присвоить:
   - `severity` (`info/warning/blocking`),
   - `applies_to` (форма/растворитель/вещество),
   - `conflict_policy` (что делать при коллизии с текущими правилами).
4. После валидации переносить в:
   - `SubstancePropertyCatalog` (если это правило по веществу),
   - профильные блоки `WaterSolutionsBlock`, `NonAqueousSolutionsBlock`, `PowdersTriturationsBlock`, `OintmentsBlock` (если это процессное правило).

## 3) Что можно внедрить первым (безопасно)
- Добавить только `info/warning`-правила (без `blocking`) из книги.
- Использовать их как "поясняющие рекомендации" в разделах `Технологія` и `Контроль якості`.
- Блокирующие правила включать только после проверки на реальных кейсах.

## 4) Минимальный roadmap
1. Очистка чанков книги для retrieval (1 день).
2. Генератор `engine_rule_candidates.json` (1 день).
3. Ручная валидация топ-50 правил (1-2 дня).
4. Интеграция в `SubstancePropertyCatalog` + 2 блока (`WaterSolutions`, `Powders`) (1-2 дня).
5. Прогон регрессии на fixture-сценариях RxEngine.
